Url과 리소스

URL(Unifomr Resources Locator) 인터넷의 리소스를 가리키는 표준 이름으로 그것이 어디에 있고 어떻게 접근할수 있는지 알려준다.

Url 문법구조_컴포넌트

<스킴>:<사용자이름>:<비밀번호>@<호스트>:<포트>/<경로>;<파라미터>?<질의>#<프래그먼트>
  • 스킴: 리소스를 가져오기위한 프로토콜
    • http: 사용자 이름과 비밀이 없는 일반적은 Url포맷 기본 포트 80 
      http://<호스트>:<포트>?<질의>?#<프래그먼트>
http://www.naver.com:80/news?name=youyoungtae
    • https: http와 유사하나 커넥션 양끝단에 ssl(보안)레이어추가 기본포트 443 
      https://<호스트>:<포트>?<질의>?#<프래그먼트>
https://www.naver.com:80/news?name=youyoungtae
    • ftp: 파일전송 프로토콜 
      ftp://<사용자이름>:<비밀번호>@<호스트>:<포트>/<경로>;<파라미터>
ftp://anonymous:pass11@www.naver.com:80/newsnum1
    • rtsp: 실시간 스트리밍 프로토콜 ,rtspu (udp계층 이용) 
      rtsp://<사용자이름>:<비밀번호>@<호스트>:<포트>/<경로>
rtsp://anonymous:pass11@www.naver.com:80/movie/아이유/라일락
    • file:머신에서 접근가능한 파일시스템
    • malito:메일
    • news:뉴스
    • telnet:텔넷 대화형 서비스
  • 사용자 이름 ,비밀번호: 사용자가 리소스를 요청할때 사용자와비밀번호를 요구하는 프로토콜에 사용
  • 호스트,포트: Url에 핵심으로 서버에 위치와 서버내에서 접속해야하는 네트워크 포트
  • 경로 : 서버내에서의 리소스 위치 , '/' 슬래쉬 문자를 구분자로 사용한다(유닉스,리눅스,맥) 경로조각은 파라미터를 가질수 있다.
  • 파라미터 : 리소스를 찾기위한 추가정보 ';'키=값 쌍으로 존재 
    ftp://pre.ai.mit/pub;type=d
  • 질의 : 리소스를 찾기위한 추가정보 '?'키=값&<키=값> 쌍으로 존재 
    http://www.naver.com:80/news?name=youyoungtae$part=n1
  • 프래그먼트 :리소스내에서 위치 값 찾는데 활용
